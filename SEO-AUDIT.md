## Audit Methodology

Run this audit monthly, or after any significant content change.

### Phase 1: Technical Health

Check these first — if the foundation is broken, content optimisation is wasted.

#### 1.1 Crawlability & Indexing

- [ ] `robots.ts` serves correct rules (check `finnishsaunaguide.com/robots.txt`)
- [ ] `sitemap.xml` lists all pages with correct URLs (check `finnishsaunaguide.com/sitemap.xml`)
- [ ] No orphan pages (every page linked from at least one other page)
- [ ] No broken internal links (`<Link href="...">` targets all resolve)
- [ ] All pages return 200 status (no 404s, no redirects chains)
- [ ] Google Search Console: check for crawl errors, excluded pages

#### 1.2 Page Speed & Core Web Vitals

- [ ] Run Lighthouse on homepage + 2 content pages
- [ ] LCP (Largest Contentful Paint) < 2.5s — hero images use `priority` and `sizes`
- [ ] INP (Interaction to Next Paint) < 200ms — minimal JS, SSG helps
- [ ] CLS (Cumulative Layout Shift) < 0.1 — images have fixed aspect ratios
- [ ] Total page weight reasonable (images are biggest factor — Next.js optimises)

#### 1.3 Mobile

- [ ] All pages responsive (test at 375px, 768px, 1024px)
- [ ] No horizontal scroll
- [ ] Tap targets at least 48px
- [ ] Font sizes readable without zooming (min 16px body)

#### 1.4 Security & Headers

- [ ] HTTPS everywhere (Vercel handles this)
- [ ] No mixed content warnings
- [ ] Canonical URLs set (Next.js handles via metadata)

### Phase 2: On-Page SEO

#### 2.1 Metadata Audit

Every page must have:

- [ ] Unique `<title>` — max 60 chars, primary keyword near the front
- [ ] Unique `<meta description>` — 150–160 chars, includes CTA language
- [ ] `keywords` array in metadata export (Next.js metadata API)
- [ ] Canonical URL (auto-generated by Next.js)

**Title formula:** `[Primary keyword] — [Qualifier or benefit] | Finnish Sauna Guide`

#### 2.2 Heading Structure

- [ ] One `<h1>` per page (never zero, never two)
- [ ] `<h2>` for major sections, `<h3>` for sub-sections
- [ ] No heading level skips (h1 → h3 without h2)
- [ ] Primary keyword in H1
- [ ] Related keywords naturally in H2s

#### 2.3 Content Quality

- [ ] Every article page > 800 words
- [ ] Hub pages have clear navigation to sub-pages
- [ ] Internal links in body copy (not just nav/footer)
- [ ] No thin pages (every page provides unique value)
- [ ] Finnish terms include English definitions on first use
- [ ] "Written by a Finn" authority signal in meta descriptions

#### 2.4 Image SEO

- [ ] All images have descriptive `alt` text (keyword-rich but natural)
- [ ] Filenames are SEO-friendly (e.g., `wool-felt-sauna-hats.jpg` not `IMG_4523.jpg`)
- [ ] Next.js `<Image>` component used (automatic WebP/AVIF, lazy loading, srcset)
- [ ] Hero images use `priority` prop for LCP
- [ ] `sizes` attribute set correctly (`100vw` for full-width, `(max-width: 768px) 100vw, 50vw` for grids)

### Phase 3: Schema Markup

| Article pages | `Article` + `BreadcrumbList` | Active |

| FAQ page | `FAQPage` + `BreadcrumbList` | Active |
| Glossary page | `DefinedTermSet` + `BreadcrumbList` | Active |

#### 3.2 Schema Validation

- [ ] Test each page with Google Rich Results Test
- [ ] No errors in JSON-LD syntax
- [ ] `@context` is `https://schema.org`
- [ ] All required properties present for each type
- [ ] FAQ schema includes all Q&A pairs (for rich snippet eligibility)

### Phase 4: Link Architecture

#### 4.1 Internal Linking

- [ ] Homepage links to all hub pages
- [ ] Hub pages link to all their sub-pages
- [ ] Sub-pages link back to their hub and to related sub-pages
- [ ] FAQ and Glossary cross-link to relevant content pages
- [ ] Footer links cover all major sections
- [ ] No page requires more than 3 clicks from homepage

#### 4.2 Affiliate Links

- [ ] All affiliate links have `rel="noopener noreferrer nofollow sponsored"`
- [ ] Amazon links include `tag=viisikanta-20`
- [ ] No placeholder `#affiliate` links remaining
- [ ] Affiliate disclosure present on pages with product links
- [ ] CTA text is "Check Price" (not "Buy Now")

### Phase 5: AI Search Optimisation (GEO)

This site allows AI crawlers. Optimise for AI-powered search (ChatGPT, Perplexity, Google AI Overviews).

- [ ] `robots.ts` allows: GPTBot, Google-Extended, ClaudeBot, PerplexityBot, Amazonbot
- [ ] Consider adding `llms.txt` at root (concise site summary for AI crawlers)
- [ ] FAQ content structured for direct AI citation (clear Q&A format)
- [ ] Glossary definitions are concise, citable sentences
- [ ] Authority signals present: "Written by a Finn", first-person experience
- [ ] Content is factual and specific (AI prefers citable facts over vague claims)

## Tools

### Free

| Tool | What For | URL |
|------|----------|-----|
| **Google Search Console** | Index coverage, crawl errors, search performance | search.google.com/search-console |
| **Google Rich Results Test** | Schema validation | search.google.com/test/rich-results |
| **PageSpeed Insights** | Core Web Vitals, Lighthouse | pagespeed.web.dev |
| **Schema.org Validator** | JSON-LD syntax check | validator.schema.org |
| **Bing Webmaster Tools** | Additional indexing data, AI search visibility | bing.com/webmasters |

### In Next.js

| Feature | How |
|---------|-----|
| Metadata | `export const metadata: Metadata` in each `page.tsx` |
| Sitemap | `src/app/sitemap.ts` — auto-generates XML |
| Robots | `src/app/robots.ts` — configurable per crawler |
| Schema | JSON-LD via `<script type="application/ld+json">` in page components |
| Images | `next/image` — automatic optimisation, WebP/AVIF, lazy loading |
| Analytics | GA4 script in `layout.tsx` |

### CLI Quick Checks

```bash
# Verify sitemap
curl -s https://finnishsaunaguide.com/sitemap.xml | head -50

# Verify robots
curl -s https://finnishsaunaguide.com/robots.txt

# Check page status codes
curl -s -o /dev/null -w "%{http_code}" https://finnishsaunaguide.com/faq

# Find pages missing metadata
grep -rL "export const metadata" src/app/*/page.tsx

# Find placeholder affiliate links
grep -r "#affiliate" src/

# Find remaining "Loyly" misspellings
grep -ri "loyly" src/ --include="*.tsx"

# Check image alt text coverage
grep -r '<Image' src/ --include="*.tsx" | grep -v 'alt='
```
